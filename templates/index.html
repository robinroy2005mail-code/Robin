UCI Cleveland Dataset Metadata

The Cleveland database is a gold standard cardiac repository comprising 303 cases. In our pre-processing, patients indicating categorical integers 1, 2, 3, or 4 for disease narrowing are classified under diagnostic Heart Disease (1), while level 0 implies Normal (0).

Heart Disease Prediction System using UCI Cleveland Dataset

ACADEMIC FINAL YEAR PROJECT DOCUMENTATION REPORT

1. Abstract

Cardiovascular diseases (CVDs) remain the leading cause of mortality globally, taking an estimated 17.9 million lives each year. Early identification of cardiac vulnerabilities is crucial for preventive healthcare. This project develops an AI-powered Clinical Decision Support System (CDSS) that leverages machine learning models to assess coronary artery disease risk. Utilizing the UCI Cleveland Heart Disease dataset, the system implements an ensemble stacking pipeline consisting of Random Forest and XGBoost classifiers optimized through a meta-logistic regression algorithm. Secure user logins (with JWT) and automated ReportLab PDF reports compose a complete medical informatics tool.

2. Problem Statement

Traditional clinical diagnostic procedures for coronary artery disease rely on intensive invasive processes like coronary angiography, which are expensive, require expert clinicians, and present localized risks. Non-invasive, machine learning-driven risk prediction serves as a critical first-pass screening utility. The core challenge is establishing high diagnostic accuracy (sensitivity and specificity) using non-invasive clinical attributes such as resting blood pressure, serum cholesterol levels, chest pain classifications, and electrocardiogram indexes.

3. Objectives

  • Implement a preprocessing pipeline handling missing data and standardizing values.
  • Train and validate Random Forest, XGBoost, and Stacking ensemble classifiers.
  • Develop a secure role-based full-stack system with JWT session security.
  • Format clinical recommendations and export downloadable PDF diagnostic metrics.
  • Establish an elegant responsive layout with clean medical illustrations.

4. System Architecture

The system operates on a client-server architecture. The frontend SPA manages state changes dynamically via custom Hash Routing to prevent full-page refreshes. It communicates with the Python Flask Backend over secure HTTP REST endpoints. JWT tokens are verified on all restricted routes. The backend manages SQL reads/writes in SQLite, executes machine learning inference using serialized joblib files, and parses PDF files on the fly.

[Client Web Browser SPA] | (HTTP Requests + JWT Bearer) v [Flask REST API Gateway] / \ v v [SQLite DB] [ML Ensemble Models] (User & Logs) (RF, XGB, Stack Scalers) | v [ReportLab Engine] --> (PDF Report)

5. Experimental Results

During cross-validation and test set assessments, models achieved high classification benchmarks. The baseline Random Forest model demonstrates robust performance with 81.9% accuracy. XGBoost displays optimized specificity, scoring 83.6% accuracy. The Stacking Meta-Classifier establishes the highest accuracy (~85.2%) with balanced sensitivity (Recall) and specificity (Precision), verifying that multi-model voting successfully overrides single-classifier bias.

6. Future Scope

Future enhancements include integrating real-time telemetry inputs from smart wearable IoT devices (tracking heart rate variability and blood pressure levels continuously), utilizing deep learning neural networks, and aligning security frameworks with HIPAA compliance protocols for deployment in clinical testbed scenarios.
